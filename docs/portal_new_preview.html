One cohort exploration workflow

Sample, variant/gene, phenotype profile로 같은 CRDC evidence를 탐색합니다

검색 시작점은 다르지만, 해석은 CRDC 내부 evidence를 먼저 보고 rare disease reference와 secondary annotation을 덧붙이는 구조입니다.

1. Search subject

Sample ID, variant/gene, HPO profile 중 하나로 시작합니다.

2. Optional HPO context

Context는 genotype similarity가 아니라 HPO phenotype context입니다.

3. Evidence layers

CRDC first → Orphanet/HPO → DECIPHER/MONDO/OMIM → secondary badges.

Our purpose

CRDC 내부에서 먼저 찾고, reference annotation은 해석을 돕는 층으로 보여줍니다

Primary CRDC evidence sample/HPO/variant recurrence, carrier group, investigator pattern
Reference Orphanet · HPO core disease/gene/HPO profile support
Secondary DECIPHER · MONDO · OMIM · PanelApp · pathway separate annotation badges, not filters
Context rule: active context가 있으면 HPO pattern overlap을 봅니다. Variant page에서도 variant 자체가 아니라 carrier HPO profile과 context HPO를 비교합니다.
Hidden goal: 사용자가 생각하지 못한 cohort/group/gene/variant insight를 찾을 수 있게 합니다.
